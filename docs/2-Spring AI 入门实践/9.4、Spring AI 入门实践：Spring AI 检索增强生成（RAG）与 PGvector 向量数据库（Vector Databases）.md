# Spring AI 入门实践：Spring AI 与 PGvector 向量数据库（Vector Databases）

> 基于 Spring AI 框架实现与 PGvector 向量数据库的集成，结合 PostgreSQL 关系型数据库和向量搜索能力，提供文档向量存储、相似度搜索、SQL查询等功能。

---

## 一、项目概述

### 1.1 项目定位

本项目是 Spring AI 框架下集成 PGvector 向量数据库的示例，展示了如何在 Java/Spring Boot 应用中结合 PostgreSQL 关系型数据库和向量搜索能力，构建 RAG 应用。

### 1.2 技术栈

| 组件 | 版本 | 说明 |
|------|------|------|
| Spring Boot | 3.5.6 | 基础框架 |
| Spring AI | 1.1.4 | AI 能力集成 |
| PostgreSQL | - | 关系型数据库 |
| PGvector | - | PostgreSQL 向量扩展 |
| spring-ai-starter-vector-store-pgvector | - | Spring AI PGvector 集成 |

### 1.3 代码地址
**GitHub**：https://github.com/partme-ai/spring-ai-examples/tree/main/spring-ai-ollama-rag-pgvector
**本地路径**：`spring-ai-ollama-rag-pgvector/`

### 1.4 核心功能

- ✅ 文档向量存储：支持将文档转换为向量并存储
- ✅ 相似度搜索：基于向量相似度检索相关文档
- ✅ SQL 查询：结合传统 SQL 查询能力
- ✅ 元数据过滤：支持基于元数据的过滤查询
- ✅ 关系型数据：利用 PostgreSQL 的关系型数据能力
- ✅ 向量索引优化：支持 IVFFlat 等索引类型

## 二、PGvector 向量数据库简介

### 2.1 PGvector 介绍

PGvector 是 PostgreSQL 数据库的一个扩展，用于存储和搜索向量数据，为 PostgreSQL 添加了向量相似度搜索能力。PGvector 让你可以在熟悉的 PostgreSQL 数据库中直接使用向量搜索，无需额外的向量数据库。

### 2.2 核心特性

| 特性 | 说明 |
|------|------|
| **PostgreSQL 扩展** | 作为 PostgreSQL 扩展安装，无缝集成 |
| **精确和近似搜索** | 支持精确搜索和 IVFFlat 近似搜索 |
| **多种距离度量** | 支持 L2、内积、余弦相似度等 |
| **SQL 集成** | 可以与 SQL 查询结合使用 |
| **ACID 事务** | 继承 PostgreSQL 的事务特性 |
| **开源免费** | PostgreSQL 开源许可证 |

### 2.3 距离度量

| 距离度量 | 说明 | 操作符 |
|---------|------|--------|
| **L2 距离** | 欧几里得距离 | `<->` |
| **内积** | 点积距离 | `<#>` |
| **余弦相似度** | 余弦距离 | `<=>` |

## 五、代码实现详解

### 5.1 PGvector 配置类

```java
package com.example.pgvector.config;

import org.springframework.ai.embedding.EmbeddingClient;
import org.springframework.ai.vectorstore.PgVectorStore;
import org.springframework.ai.vectorstore.VectorStore;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.jdbc.core.JdbcTemplate;

@Configuration
public class PgVectorConfig {

    @Value("${spring.ai.vectorstore.pgvector.table-name:documents}")
    private String tableName;

    @Value("${spring.ai.vectorstore.pgvector.vector-column:embedding}")
    private String vectorColumn;

    @Value("${spring.ai.vectorstore.pgvector.content-column:content}")
    private String contentColumn;

    @Value("${spring.ai.vectorstore.pgvector.metadata-column:metadata}")
    private String metadataColumn;

    @Bean
    public VectorStore vectorStore(EmbeddingClient embeddingClient, JdbcTemplate jdbcTemplate) {
        return new PgVectorStore(
            embeddingClient,
            jdbcTemplate,
            tableName,
            vectorColumn,
            contentColumn,
            metadataColumn
        );
    }
}
```

### 5.2 向量数据库服务类

```java
package com.example.pgvector.service;

import org.springframework.ai.document.Document;
import org.springframework.ai.vectorstore.SearchRequest;
import org.springframework.ai.vectorstore.VectorStore;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.jdbc.core.JdbcTemplate;
import org.springframework.stereotype.Service;

import java.util.List;
import java.util.Map;

@Service
public class PgVectorService {

    @Autowired
    private VectorStore vectorStore;

    @Autowired
    private JdbcTemplate jdbcTemplate;

    public void addDocuments(List<Document> documents) {
        vectorStore.add(documents);
    }

    public List<Document> similaritySearch(String query, int topK) {
        return vectorStore.similaritySearch(query, topK);
    }

    public List<Document> similaritySearchWithFilter(String query, int topK, Map<String, Object> filter) {
        SearchRequest request = SearchRequest.query(query)
            .withTopK(topK)
            .withSimilarityThreshold(0.0)
            .withFilterExpression(filter);
        return vectorStore.similaritySearch(request);
    }

    public void deleteDocuments(List<String> ids) {
        vectorStore.delete(ids);
    }

    public void createVectorIndex(String indexName, String tableName, String columnName, int lists) {
        String sql = String.format(
            "CREATE INDEX IF NOT EXISTS %s ON %s USING ivfflat (%s vector_cosine_ops) WITH (lists = %d)",
            indexName, tableName, columnName, lists
        );
        jdbcTemplate.execute(sql);
    }
}
```

### 5.3 REST API 控制器

```java
package com.example.pgvector.controller;

import org.springframework.ai.document.Document;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;
import com.example.pgvector.service.PgVectorService;

import java.util.ArrayList;
import java.util.List;
import java.util.Map;

@RestController
@RequestMapping("/api/pgvector")
public class PgVectorController {

    @Autowired
    private PgVectorService pgVectorService;

    @PostMapping("/documents")
    public String addDocuments(@RequestBody List<Map<String, Object>> docs) {
        List<Document> documents = new ArrayList<>();
        for (Map<String, Object> doc : docs) {
            String content = (String) doc.get("content");
            @SuppressWarnings("unchecked")
            Map<String, Object> metadata = (Map<String, Object>) doc.get("metadata");
            documents.add(new Document(content, metadata));
        }
        pgVectorService.addDocuments(documents);
        return "Documents added successfully";
    }

    @GetMapping("/search")
    public List<Document> search(
            @RequestParam String query,
            @RequestParam(defaultValue = "5") int topK) {
        return pgVectorService.similaritySearch(query, topK);
    }

    @PostMapping("/search/filter")
    public List<Document> searchWithFilter(
            @RequestParam String query,
            @RequestParam(defaultValue = "5") int topK,
            @RequestBody Map<String, Object> filter) {
        return pgVectorService.similaritySearchWithFilter(query, topK, filter);
    }

    @DeleteMapping("/documents")
    public String deleteDocuments(@RequestBody List<String> ids) {
        pgVectorService.deleteDocuments(ids);
        return "Documents deleted successfully";
    }

    @PostMapping("/index")
    public String createIndex(@RequestBody Map<String, Object> request) {
        String indexName = (String) request.getOrDefault("indexName", "idx_embedding");
        String tableName = (String) request.getOrDefault("tableName", "documents");
        String columnName = (String) request.getOrDefault("columnName", "embedding");
        int lists = (int) request.getOrDefault("lists", 100);
        pgVectorService.createVectorIndex(indexName, tableName, columnName, lists);
        return "Vector index created successfully";
    }
}
```

### 8.2 Java 客户端

```java
import org.springframework.web.client.RestTemplate;
import org.springframework.http.HttpEntity;
import org.springframework.http.HttpHeaders;
import org.springframework.http.MediaType;
import java.util.List;
import java.util.Map;

public class PgVectorClient {

    private static final String BASE_URL = "http://localhost:8080/api/pgvector";
    private RestTemplate restTemplate = new RestTemplate();

    public void addDocuments(List<Map<String, Object>> docs) {
        HttpHeaders headers = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_JSON);
        HttpEntity<List<Map<String, Object>>> request = new HttpEntity<>(docs, headers);
        restTemplate.postForObject(BASE_URL + "/documents", request, String.class);
    }

    public List<Map<String, Object>> search(String query, int topK) {
        String url = BASE_URL + "/search?query={query}&topK={topK}";
        return restTemplate.getForObject(url, List.class, query, topK);
    }
}
```

## 十、常见问题

### Q1: vector 扩展不存在？

确保在数据库中执行了 `CREATE EXTENSION IF NOT EXISTS vector;`。可以使用 `\dx` 命令在 psql 中查看已安装的扩展。

### Q2: 如何选择合适的 lists 参数？

lists 参数控制 IVFFlat 索引的聚类数量。一般来说：
- 小规模数据（< 100万）：lists = 100
- 中等规模（100万-1000万）：lists = 1000
- 大规模数据（> 1000万）：lists = 10000

### Q3: 精确搜索和近似搜索的区别？

- 精确搜索：不使用索引，100% 召回率，但速度较慢
- 近似搜索（IVFFlat）：使用索引，速度快，但有一定精度损失
