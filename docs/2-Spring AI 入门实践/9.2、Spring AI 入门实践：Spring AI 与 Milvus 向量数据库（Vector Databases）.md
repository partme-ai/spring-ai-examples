# Spring AI 入门实践：Spring AI 与 Milvus 向量数据库（Vector Databases）

> 基于 Spring AI 框架实现与 Milvus 向量数据库的集成，提供高性能向量存储、相似度搜索、索引优化等功能，支持海量向量数据的存储和检索。

---

## 一、项目概述

### 1.1 项目定位

本项目是 Spring AI 框架下集成 Milvus 向量数据库的示例，展示了如何在 Java/Spring Boot 应用中实现高性能的向量存储、相似度检索和索引优化功能。

### 1.2 技术栈

| 组件 | 版本 | 说明 |
|------|------|------|
| Spring Boot | 3.5.6 | 基础框架 |
| Spring AI | 1.1.4 | AI 能力集成 |
| Milvus | 2.3.0+ | 向量数据库 |
| spring-ai-starter-vector-store-milvus | - | Spring AI Milvus 集成 |

### 1.3 核心功能

- ✅ 高性能向量存储：支持海量向量数据的存储
- ✅ 相似度搜索：基于向量相似度快速检索
- ✅ 索引优化：支持多种索引类型（IVF_FLAT、IVF_SQ8 等）
- ✅ 距离度量：支持 L2、IP 等多种距离度量方式
- ✅ 元数据过滤：支持基于元数据的过滤查询
- ✅ 批量操作：支持批量添加、删除文档

## 二、Milvus 向量数据库简介

### 2.1 Milvus 介绍

Milvus 是一个高性能、可扩展的向量数据库，专为 AI 应用设计，支持海量向量数据的存储和检索。Milvus 提供多种索引类型和距离度量方式，能够满足不同场景下的向量检索需求。

### 2.2 核心特性

| 特性 | 说明 |
|------|------|
| **高性能** | 支持十亿级向量数据的毫秒级检索 |
| **可扩展** | 分布式架构，支持水平扩展 |
| **多索引类型** | 支持 IVF_FLAT、IVF_SQ8、HNSW 等多种索引 |
| **多距离度量** | 支持 L2、IP、Cosine 等多种距离度量 |
| **持久化存储** | 支持数据持久化和高可用 |
| **云原生** | 支持 Kubernetes 部署 |

### 2.3 索引类型

| 索引类型 | 说明 | 适用场景 |
|---------|------|---------|
| **FLAT** | 暴力搜索，100% 召回率 | 小规模数据集、高召回率要求 |
| **IVF_FLAT** | 倒排文件，向量量化 | 大规模数据集，平衡精度和速度 |
| **IVF_SQ8** | 标量量化，压缩存储 | 超大规模数据集，追求查询速度 |
| **HNSW** | 图索引，高查询性能 | 对查询速度要求极高的场景 |

## 五、代码实现详解

### 5.1 Milvus 配置类

```java
package com.example.milvus.config;

import org.springframework.ai.embedding.EmbeddingClient;
import org.springframework.ai.vectorstore.MilvusVectorStore;
import org.springframework.ai.vectorstore.MilvusVectorStoreConfig;
import org.springframework.ai.vectorstore.VectorStore;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class MilvusConfig {

    @Value("${spring.ai.vectorstore.milvus.host:localhost}")
    private String host;

    @Value("${spring.ai.vectorstore.milvus.port:19530}")
    private int port;

    @Value("${spring.ai.vectorstore.milvus.collection-name:my-collection}")
    private String collectionName;

    @Value("${spring.ai.vectorstore.milvus.index-type:IVF_FLAT}")
    private String indexType;

    @Value("${spring.ai.vectorstore.milvus.metric-type:L2}")
    private String metricType;

    @Value("${spring.ai.vectorstore.milvus.dimension:1536}")
    private int dimension;

    @Bean
    public VectorStore vectorStore(EmbeddingClient embeddingClient) {
        MilvusVectorStoreConfig config = MilvusVectorStoreConfig.builder()
            .withHost(host)
            .withPort(port)
            .withCollectionName(collectionName)
            .withIndexType(indexType)
            .withMetricType(metricType)
            .withDimension(dimension)
            .build();

        return new MilvusVectorStore(embeddingClient, config);
    }
}
```

### 5.2 向量数据库服务类

```java
package com.example.milvus.service;

import org.springframework.ai.document.Document;
import org.springframework.ai.vectorstore.SearchRequest;
import org.springframework.ai.vectorstore.VectorStore;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

import java.util.List;
import java.util.Map;

@Service
public class MilvusService {

    @Autowired
    private VectorStore vectorStore;

    public void addDocuments(List<Document> documents) {
        vectorStore.add(documents);
    }

    public List<Document> similaritySearch(String query, int topK) {
        return vectorStore.similaritySearch(query, topK);
    }

    public List<Document> similaritySearchWithFilter(String query, int topK, Map<String, Object> filter) {
        SearchRequest request = SearchRequest.query(query)
            .withTopK(topK)
            .withSimilarityThreshold(0.0)
            .withFilterExpression(filter);
        return vectorStore.similaritySearch(request);
    }

    public void deleteDocuments(List<String> ids) {
        vectorStore.delete(ids);
    }
}
```

### 5.3 REST API 控制器

```java
package com.example.milvus.controller;

import org.springframework.ai.document.Document;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;
import com.example.milvus.service.MilvusService;

import java.util.ArrayList;
import java.util.List;
import java.util.Map;

@RestController
@RequestMapping("/api/milvus")
public class MilvusController {

    @Autowired
    private MilvusService milvusService;

    @PostMapping("/documents")
    public String addDocuments(@RequestBody List<Map<String, Object>> docs) {
        List<Document> documents = new ArrayList<>();
        for (Map<String, Object> doc : docs) {
            String content = (String) doc.get("content");
            @SuppressWarnings("unchecked")
            Map<String, Object> metadata = (Map<String, Object>) doc.get("metadata");
            documents.add(new Document(content, metadata));
        }
        milvusService.addDocuments(documents);
        return "Documents added successfully";
    }

    @GetMapping("/search")
    public List<Document> search(
            @RequestParam String query,
            @RequestParam(defaultValue = "5") int topK) {
        return milvusService.similaritySearch(query, topK);
    }

    @PostMapping("/search/filter")
    public List<Document> searchWithFilter(
            @RequestParam String query,
            @RequestParam(defaultValue = "5") int topK,
            @RequestBody Map<String, Object> filter) {
        return milvusService.similaritySearchWithFilter(query, topK, filter);
    }

    @DeleteMapping("/documents")
    public String deleteDocuments(@RequestBody List<String> ids) {
        milvusService.deleteDocuments(ids);
        return "Documents deleted successfully";
    }
}
```

```java
import org.springframework.web.client.RestTemplate;
import org.springframework.http.HttpEntity;
import org.springframework.http.HttpHeaders;
import org.springframework.http.MediaType;
import java.util.List;
import java.util.Map;

public class MilvusClient {

    private static final String BASE_URL = "http://localhost:8080/api/milvus";
    private RestTemplate restTemplate = new RestTemplate();

    public void addDocuments(List<Map<String, Object>> docs) {
        HttpHeaders headers = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_JSON);
        HttpEntity<List<Map<String, Object>>> request = new HttpEntity<>(docs, headers);
        restTemplate.postForObject(BASE_URL + "/documents", request, String.class);
    }

    public List<Map<String, Object>> search(String query, int topK) {
        String url = BASE_URL + "/search?query={query}&topK={topK}";
        return restTemplate.getForObject(url, List.class, query, topK);
    }
}
```

## 十、常见问题

### Q1: Milvus 服务连接失败？

检查 Milvus 服务是否正常启动，确认配置的 host 和 port 是否正确。可以通过 `curl http://localhost:9091/healthz` 验证服务状态。

### Q2: 如何选择合适的索引类型？

- 小规模数据集（< 100万）：使用 FLAT，保证 100% 召回率
- 中等规模数据集（100万-1000万）：使用 IVF_FLAT，平衡精度和速度
- 大规模数据集（> 1000万）：使用 IVF_SQ8 或 HNSW，追求查询速度

### Q3: 如何优化查询性能？

- 选择合适的索引类型
- 调整索引参数（如 IVF 的 nlist）
- 使用批量查询
- 合理设置 topK 参数
