# Spring AI 入门实践：Spring AI 与 Qdrant 向量数据库（Vector Databases）

> 基于 Spring AI 框架实现与 Qdrant 向量数据库的集成，使用高性能的开源向量搜索引擎，提供近似最近邻搜索、元数据过滤和分布式部署。

---

## 一、项目概述

### 1.1 项目定位

本项目是 Spring AI 框架下集成 Qdrant 向量数据库的示例，展示了如何在 Java/Spring Boot 应用中使用高性能的 Qdrant 开源向量搜索引擎，构建 RAG 应用。

### 1.2 技术栈

| 组件 | 版本 | 说明 |
|------|------|------|
| Spring Boot | 3.5.6 | 基础框架 |
| Spring AI | 1.1.4 | AI 能力集成 |
| Qdrant | - | 开源向量搜索引擎 |
| spring-ai-starter-vector-store-qdrant | - | Spring AI Qdrant 集成 |

### 1.3 核心功能

- ✅ 高性能搜索：专为大规模向量数据优化
- ✅ 近似最近邻搜索：HNSW 算法支持
- ✅ 元数据过滤：强大的元数据过滤查询
- ✅ 分布式部署：支持集群模式
- ✅ 持久化存储：向量数据持久化到磁盘
- ✅ REST API：简单易用的 REST 接口
- ✅ 多种距离度量：支持余弦、点积、欧几里得等

## 二、Qdrant 向量数据库简介

### 2.1 Qdrant 介绍

Qdrant 是一个高性能的向量搜索引擎，专为 AI 应用设计，提供向量相似度搜索和过滤功能。Qdrant 采用 Rust 编写，性能优异，支持 HNSW 等近似最近邻搜索算法，适合大规模向量数据场景。

### 2.2 核心特性

| 特性 | 说明 |
|------|------|
| **高性能** | Rust 编写，专为大规模向量数据优化 |
| **HNSW 算法** | 支持高效的近似最近邻搜索 |
| **元数据过滤** | 强大的元数据过滤和筛选能力 |
| **分布式部署** | 支持集群模式，水平扩展 |
| **持久化存储** | 向量数据持久化到磁盘 |
| **REST API** | 简单易用的 REST 接口 |
| **多种距离度量** | 支持余弦、点积、欧几里得等 |
| **开源免费** | Apache 2.0 许可证 |

### 2.3 距离度量

Qdrant 支持多种距离度量：

| 距离度量 | 说明 |
|---------|------|
| **Cosine** | 余弦相似度（默认） |
| **Dot** | 点积距离 |
| **Euclid** | 欧几里得距离 |

## 五、代码实现详解

### 5.1 Qdrant 配置类

```java
package com.example.qdrant.config;

import org.springframework.ai.embedding.EmbeddingClient;
import org.springframework.ai.vectorstore.QdrantVectorStore;
import org.springframework.ai.vectorstore.QdrantVectorStoreConfig;
import org.springframework.ai.vectorstore.VectorStore;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class QdrantConfig {

    @Value("${spring.ai.vectorstore.qdrant.url}")
    private String url;

    @Value("${spring.ai.vectorstore.qdrant.collection-name}")
    private String collectionName;

    @Bean
    public VectorStore vectorStore(EmbeddingClient embeddingClient) {
        QdrantVectorStoreConfig config = QdrantVectorStoreConfig.builder()
            .withUrl(url)
            .withCollectionName(collectionName)
            .withDistanceFunction(QdrantVectorStore.DistanceFunction.COSINE)
            .withVectorSize(1536)
            .build();
        
        return new QdrantVectorStore(embeddingClient, config);
    }
}
```

### 5.2 向量数据库服务类

```java
package com.example.qdrant.service;

import org.springframework.ai.document.Document;
import org.springframework.ai.vectorstore.SearchRequest;
import org.springframework.ai.vectorstore.VectorStore;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

import java.util.List;
import java.util.Map;

@Service
public class QdrantService {

    @Autowired
    private VectorStore vectorStore;

    public void addDocuments(List<Document> documents) {
        vectorStore.add(documents);
    }

    public List<Document> similaritySearch(String query, int topK) {
        return vectorStore.similaritySearch(query, topK);
    }

    public List<Document> similaritySearchWithFilter(String query, int topK, Map<String, Object> filter) {
        SearchRequest request = SearchRequest.query(query)
            .withTopK(topK)
            .withSimilarityThreshold(0.0)
            .withFilterExpression(filter);
        return vectorStore.similaritySearch(request);
    }

    public void deleteDocuments(List<String> ids) {
        vectorStore.delete(ids);
    }
}
```

### 5.3 REST API 控制器

```java
package com.example.qdrant.controller;

import org.springframework.ai.document.Document;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;
import com.example.qdrant.service.QdrantService;

import java.util.ArrayList;
import java.util.List;
import java.util.Map;

@RestController
@RequestMapping("/api/qdrant")
public class QdrantController {

    @Autowired
    private QdrantService qdrantService;

    @PostMapping("/documents")
    public String addDocuments(@RequestBody List<Map<String, Object>> docs) {
        List<Document> documents = new ArrayList<>();
        for (Map<String, Object> doc : docs) {
            String content = (String) doc.get("content");
            @SuppressWarnings("unchecked")
            Map<String, Object> metadata = (Map<String, Object>) doc.get("metadata");
            documents.add(new Document(content, metadata));
        }
        qdrantService.addDocuments(documents);
        return "Documents added successfully";
    }

    @GetMapping("/search")
    public List<Document> search(
            @RequestParam String query,
            @RequestParam(defaultValue = "5") int topK) {
        return qdrantService.similaritySearch(query, topK);
    }

    @PostMapping("/search/filter")
    public List<Document> searchWithFilter(
            @RequestParam String query,
            @RequestParam(defaultValue = "5") int topK,
            @RequestBody Map<String, Object> filter) {
        return qdrantService.similaritySearchWithFilter(query, topK, filter);
    }

    @DeleteMapping("/documents")
    public String deleteDocuments(@RequestBody List<String> ids) {
        qdrantService.deleteDocuments(ids);
        return "Documents deleted successfully";
    }
}
```

## 十、常见问题

### Q1: 如何选择合适的 HNSW 配置？

HNSW 的主要配置参数包括：
- `ef_construction`：构建索引时的探索范围（建议 128-512）
- `m`：每个节点的连接数（建议 16-64）
- `ef_search`：搜索时的探索范围（建议 32-256）

### Q2: Qdrant 的集群如何配置？

Qdrant 支持分布式部署，可以通过配置多个节点来实现高可用性和水平扩展。详细配置请参考 Qdrant 官方文档。

### Q3: 向量数据如何备份？

Qdrant 支持集合快照功能，可以通过 REST API 创建和恢复快照。
