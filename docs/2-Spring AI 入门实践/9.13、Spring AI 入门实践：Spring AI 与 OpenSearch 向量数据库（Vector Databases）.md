# 9.13、Spring AI 入门实践：Spring AI 与 OpenSearch 向量数据库（Vector Databases）

## 概述

OpenSearch 是一个开源的搜索和分析套件，由 AWS 基于 Elasticsearch 7.10 分支开发。OpenSearch 从 2.0 版本开始原生支持向量搜索功能，提供了 k-NN 插件来实现高效的向量检索。Spring AI 提供了与 OpenSearch 的深度集成，支持将 OpenSearch 作为向量数据库使用，实现文档的向量存储和相似度搜索。

本示例项目展示了如何结合 Spring AI 的 Ollama 嵌入模型与 OpenSearch 向量存储，构建完整的 RAG（检索增强生成）应用。项目位于 `spring-ai-ollama-rag-opensearch`。

## 准备工作

### 1. 环境要求

- **Java 17+**
- **Maven 3.8+**
- **Ollama**（用于本地运行大语言模型和嵌入模型）
- **OpenSearch 2.0+**（支持 k-NN 向量搜索）

### 2. 安装和启动 OpenSearch

可以通过 Docker 快速启动 OpenSearch：

```bash
docker run -d --name opensearch \
  -p 9200:9200 \
  -p 9600:9600 \
  -e "discovery.type=single-node" \
  -e "DISABLE_SECURITY_PLUGIN=true" \
  -e "OPENSEARCH_JAVA_OPTS=-Xms512m -Xmx512m" \
  opensearchproject/opensearch:2.12.0
```

确认 OpenSearch 已启动：

```bash
curl http://localhost:9200
```

### 3. 安装和启动 Ollama

参考 [Spring AI 与 Ollama 文本生成](./10、Spring AI 入门实践：Spring AI 使用 Ollama Chat.md) 文档安装 Ollama，并确保已拉取所需的模型：

```bash
# 拉取对话模型
ollama pull qwen3.5

# 拉取嵌入模型
ollama pull mistral
```

## 完整示例

### 向量存储基本操作

以下示例展示了使用 VectorStore 进行文档存储和相似度搜索：

```java
import org.springframework.ai.document.Document;
import org.springframework.ai.vectorstore.SearchRequest;
import org.springframework.ai.vectorstore.VectorStore;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

import java.util.List;
import java.util.Map;

@Service
public class VectorStoreService {

    @Autowired
    private VectorStore vectorStore;

    public void addDocuments() {
        List<Document> documents = List.of(
            new Document("OpenSearch 是一个开源搜索引擎", Map.of("category", "search")),
            new Document("Spring AI 提供了 AI 能力集成", Map.of("category", "ai")),
            new Document("k-NN 插件支持向量搜索", Map.of("category", "vector")),
            new Document("RAG 技术结合了检索和生成", Map.of("category", "ai"))
        );
        
        vectorStore.add(documents);
        System.out.println("文档添加完成");
    }

    public List<Document> searchDocuments(String query) {
        return vectorStore.similaritySearch(
            SearchRequest.builder()
                .query(query)
                .topK(3)
                .build()
        );
    }
}
```

### 创建 k-NN 索引

在使用向量搜索前，需要创建带有 k-NN 字段的索引：

```json
PUT /spring_ai_vectors
{
  "settings": {
    "index": {
      "knn": true,
      "knn.algo_param.ef_search": 100
    }
  },
  "mappings": {
    "properties": {
      "embedding": {
        "type": "knn_vector",
        "dimension": 768,
        "method": {
          "name": "hnsw",
          "space_type": "cosinesimil",
          "engine": "nmslib"
        }
      },
      "content": {
        "type": "text"
      }
    }
  }
}
```

### RAG 实现示例

以下是一个完整的 RAG 实现示例：

```java
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.chat.messages.UserMessage;
import org.springframework.ai.chat.prompt.Prompt;
import org.springframework.ai.chat.prompt.PromptTemplate;
import org.springframework.ai.chat.prompt.SystemPromptTemplate;
import org.springframework.ai.document.Document;
import org.springframework.ai.vectorstore.SearchRequest;
import org.springframework.ai.vectorstore.VectorStore;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;

import java.util.List;
import java.util.Map;
import java.util.stream.Collectors;

@RestController
class RAGController {
    private final ChatClient chatModel;
    private final VectorStore vectorStore;

    RAGController(ChatClient chatModel, VectorStore vectorStore) {
        this.chatModel = chatModel;
        this.vectorStore = vectorStore;
    }

    @GetMapping("/ai/rag/search")
    String searchWithRag(@RequestParam String query) {
        // 步骤1：使用自然语言查询 VectorStore
        List<Document> similarDocuments = vectorStore.similaritySearch(
                SearchRequest.builder()
                    .query(query)
                    .topK(3)
                    .build());
        
        // 步骤2：将相似文档内容拼接为上下文信息
        String context = similarDocuments.stream()
                .map(Document::getText)
                .collect(Collectors.joining("\n\n"));

        // 步骤3：构建 System Prompt
        var systemPromptTemplate = new SystemPromptTemplate("""
              你是一个有帮助的助手。
              请根据以下信息回答用户的问题：
              {context}
              """);
        var systemMessage = systemPromptTemplate.createMessage(
                Map.of("context", context));

        // 步骤4：构建用户消息
        var userMessage = new UserMessage(query);

        // 步骤5：发送 Prompt 并获取响应
        var prompt = new Prompt(List.of(systemMessage, userMessage));
        return chatModel.prompt(prompt).call().content();
    }
}
```

## 最佳实践

### 1. k-NN 算法选择

OpenSearch 支持多种 k-NN 算法：

```json
{
  "method": {
    "name": "hnsw",
    "space_type": "cosinesimil",
    "engine": "nmslib",
    "parameters": {
      "ef_construction": 128,
      "m": 24
    }
  }
}
```

- **hnsw**：适合大规模数据，查询速度快
- **ivf**：适合中等规模数据，内存占用较低

### 2. 索引设置优化

```json
{
  "settings": {
    "index": {
      "knn": true,
      "knn.algo_param.ef_search": 100,
      "number_of_shards": 3,
      "number_of_replicas": 1
    }
  }
}
```

### 3. 批量操作优化

对于大量文档的添加，建议使用批量操作：

```java
public void addDocumentsBatch(List<Document> documents) {
    // 分批处理，每批 100 条
    List<List<Document>> batches = ListUtils.partition(documents, 100);
    for (List<Document> batch : batches) {
        vectorStore.add(batch);
    }
}
```

### 4. 混合搜索

OpenSearch 支持向量搜索与传统搜索的结合：

```java
SearchRequest searchRequest = SearchRequest.builder()
    .query(query)
    .topK(10)
    .filterExpression("category:ai AND status:published")
    .build();
```

## 故障排除

### 常见问题

1. **无法连接 OpenSearch**
   - 检查 OpenSearch 服务是否启动
   - 确认端口 9200 可访问
   - 验证用户名和密码配置

2. **k-NN 索引未生效**
   - 确保使用 OpenSearch 2.0 或更高版本
   - 检查 k-NN 插件是否已安装
   - 验证索引映射配置正确

3. **嵌入失败**
   - 确认 Ollama 已拉取配置的嵌入模型
   - 检查 Ollama 服务是否正常运行
   - 验证模型名称是否正确

4. **搜索性能问题**
   - 调整 `ef_search` 参数
   - 优化索引分片配置
   - 考虑使用更高效的 k-NN 算法
