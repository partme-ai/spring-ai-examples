# 9.16、Spring AI 入门实践：Spring AI 与 Oracle 向量数据库（Vector Databases）

## 概述

Oracle Database 是企业级关系型数据库管理系统，从 Oracle Database 23c 开始原生支持向量数据类型和向量搜索功能。Oracle AI Vector Search 允许在 Oracle 数据库中存储和检索向量嵌入，结合传统 SQL 查询能力，为企业应用提供强大的 RAG 支持。Spring AI 提供了与 Oracle 的深度集成，支持将 Oracle 作为向量数据库使用，实现文档的向量存储和相似度搜索。

本示例项目展示了如何结合 Spring AI 的 Ollama 嵌入模型与 Oracle 向量存储，构建完整的 RAG（检索增强生成）应用。项目位于 `spring-ai-ollama-rag-oracle`。

## 准备工作

### 1. 环境要求

- **Java 17+**
- **Maven 3.8+**
- **Ollama**（用于本地运行大语言模型和嵌入模型）
- **Oracle Database 23c+**（支持 AI Vector Search）

### 2. 安装和启动 Oracle

可以通过 Docker 快速启动 Oracle Database：

```bash
docker run -d --name oracle \
  -p 1521:1521 \
  -e ORACLE_PWD=password \
  gvenzl/oracle-free:23-slim
```

确认 Oracle 已启动：

```bash
docker exec -it oracle sqlplus system/password@localhost:1521/freepdb1
```

### 3. 安装和启动 Ollama

参考 [Spring AI 与 Ollama 文本生成](./10、Spring AI 入门实践：Spring AI 使用 Ollama Chat.md) 文档安装 Ollama，并确保已拉取所需的模型：

```bash
# 拉取对话模型
ollama pull qwen3.5

# 拉取嵌入模型
ollama pull mistral
```

## 完整示例

### 创建向量表

Oracle 需要创建带有向量列的表：

```sql
CREATE TABLE vector_store (
    id VARCHAR2(36) PRIMARY KEY,
    content CLOB,
    metadata JSON,
    embedding VECTOR(768, FLOAT32)
);

CREATE VECTOR INDEX idx_embedding ON vector_store(embedding);
```

### 向量存储基本操作

以下示例展示了使用 VectorStore 进行文档存储和相似度搜索：

```java
import org.springframework.ai.document.Document;
import org.springframework.ai.vectorstore.SearchRequest;
import org.springframework.ai.vectorstore.VectorStore;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

import java.util.List;
import java.util.Map;

@Service
public class VectorStoreService {

    @Autowired
    private VectorStore vectorStore;

    public void addDocuments() {
        List<Document> documents = List.of(
            new Document("Oracle 是企业级数据库", Map.of("category", "database")),
            new Document("Spring AI 提供了 AI 能力集成", Map.of("category", "ai")),
            new Document("AI Vector Search 支持向量搜索", Map.of("category", "vector")),
            new Document("RAG 技术结合了检索和生成", Map.of("category", "ai"))
        );
        
        vectorStore.add(documents);
        System.out.println("文档添加完成");
    }

    public List<Document> searchDocuments(String query) {
        return vectorStore.similaritySearch(
            SearchRequest.builder()
                .query(query)
                .topK(3)
                .build()
        );
    }
}
```

### RAG 实现示例

以下是一个完整的 RAG 实现示例：

```java
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.chat.messages.UserMessage;
import org.springframework.ai.chat.prompt.Prompt;
import org.springframework.ai.chat.prompt.PromptTemplate;
import org.springframework.ai.chat.prompt.SystemPromptTemplate;
import org.springframework.ai.document.Document;
import org.springframework.ai.vectorstore.SearchRequest;
import org.springframework.ai.vectorstore.VectorStore;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;

import java.util.List;
import java.util.Map;
import java.util.stream.Collectors;

@RestController
class RAGController {
    private final ChatClient chatModel;
    private final VectorStore vectorStore;

    RAGController(ChatClient chatModel, VectorStore vectorStore) {
        this.chatModel = chatModel;
        this.vectorStore = vectorStore;
    }

    @GetMapping("/ai/rag/search")
    String searchWithRag(@RequestParam String query) {
        // 步骤1：使用自然语言查询 VectorStore
        List<Document> similarDocuments = vectorStore.similaritySearch(
                SearchRequest.builder()
                    .query(query)
                    .topK(3)
                    .build());
        
        // 步骤2：将相似文档内容拼接为上下文信息
        String context = similarDocuments.stream()
                .map(Document::getText)
                .collect(Collectors.joining("\n\n"));

        // 步骤3：构建 System Prompt
        var systemPromptTemplate = new SystemPromptTemplate("""
              你是一个有帮助的助手。
              请根据以下信息回答用户的问题：
              {context}
              """);
        var systemMessage = systemPromptTemplate.createMessage(
                Map.of("context", context));

        // 步骤4：构建用户消息
        var userMessage = new UserMessage(query);

        // 步骤5：发送 Prompt 并获取响应
        var prompt = new Prompt(List.of(systemMessage, userMessage));
        return chatModel.prompt(prompt).call().content();
    }
}
```

## 最佳实践

### 1. 向量索引优化

创建高效的向量索引：

```sql
CREATE VECTOR INDEX idx_embedding ON vector_store(embedding)
WITH TARGET ACCURACY 95
PARAMETERS (type = IVF, neighbor partitions = 10);
```

### 2. 混合查询

Oracle 支持向量搜索与传统 SQL 查询的结合：

```sql
SELECT id, content,
       VECTOR_DISTANCE(embedding, :query_vector, COSINE) AS distance
FROM vector_store
WHERE JSON_VALUE(metadata, '$.category') = 'ai'
ORDER BY distance
FETCH FIRST 10 ROWS ONLY;
```

### 3. 批量操作优化

对于大量文档的添加，建议使用批量操作：

```java
public void addDocumentsBatch(List<Document> documents) {
    // 分批处理，每批 100 条
    List<List<Document>> batches = ListUtils.partition(documents, 100);
    for (List<Document> batch : batches) {
        vectorStore.add(batch);
    }
}
```

### 4. 分区策略

对于大规模数据，考虑使用表分区：

```sql
CREATE TABLE vector_store (
    id VARCHAR2(36),
    content CLOB,
    metadata JSON,
    embedding VECTOR(768, FLOAT32),
    created_at TIMESTAMP
)
PARTITION BY RANGE (created_at) (
    PARTITION p_2024 VALUES LESS THAN (TO_DATE('2025-01-01', 'YYYY-MM-DD')),
    PARTITION p_2025 VALUES LESS THAN (TO_DATE('2026-01-01', 'YYYY-MM-DD'))
);
```

## 故障排除

### 常见问题

1. **无法连接 Oracle**
   - 检查 Oracle 服务是否启动
   - 确认端口 1521 可访问
   - 验证用户名和密码配置

2. **向量类型不支持**
   - 确保使用 Oracle Database 23c 或更高版本
   - 检查表结构是否正确创建
   - 验证向量维度配置

3. **嵌入失败**
   - 确认 Ollama 已拉取配置的嵌入模型
   - 检查 Ollama 服务是否正常运行
   - 验证模型名称是否正确

4. **搜索性能问题**
   - 创建向量索引
   - 优化查询语句
   - 考虑使用分区表策略
