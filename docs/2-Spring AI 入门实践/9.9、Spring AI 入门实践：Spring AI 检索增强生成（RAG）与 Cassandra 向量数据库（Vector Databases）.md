# 9.9、Spring AI 入门实践：Spring AI 与 Cassandra 向量数据库（Vector Databases）

## 概述

Apache Cassandra 是一个高度可扩展的分布式 NoSQL 数据库，最初由 Facebook 开发，现已广泛应用于大规模数据存储场景。Spring AI 提供了与 Cassandra 的深度集成，支持将 Cassandra 作为向量数据库使用，实现文档的向量存储和相似度搜索。

本示例项目展示了如何结合 Spring AI 的 Ollama 嵌入模型与 Cassandra 向量存储，构建完整的 RAG（检索增强生成）应用。项目位于 `spring-ai-ollama-rag-cassandra`。

## 准备工作

### 1. 环境要求

- **Java 17+**
- **Maven 3.8+**
- **Ollama**（用于本地运行大语言模型和嵌入模型）
- **Apache Cassandra**（作为向量数据库）

### 2. 安装和启动 Cassandra

可以通过 Docker 快速启动 Cassandra：

```bash
docker run -d --name cassandra \
  -p 9042:9042 \
  cassandra:latest
```

确认 Cassandra 已启动：

```bash
docker exec -it cassandra cqlsh
```

### 3. 安装和启动 Ollama

参考 [Spring AI 与 Ollama 文本生成](./10、Spring AI 入门实践：Spring AI 使用 Ollama Chat.md) 文档安装 Ollama，并确保已拉取所需的模型：

```bash
# 拉取对话模型
ollama pull qwen3.5

# 拉取嵌入模型
ollama pull mistral
```

## 完整示例

### RAG 控制器实现

以下示例展示了完整的 RAG 实现流程：

```java
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.chat.messages.UserMessage;
import org.springframework.ai.chat.prompt.Prompt;
import org.springframework.ai.chat.prompt.PromptTemplate;
import org.springframework.ai.chat.prompt.SystemPromptTemplate;
import org.springframework.ai.document.Document;
import org.springframework.ai.converter.BeanOutputConverter;
import org.springframework.ai.vectorstore.SearchRequest;
import org.springframework.ai.vectorstore.VectorStore;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;

import java.time.LocalDate;
import java.util.List;
import java.util.Map;
import java.util.stream.Collectors;

@RestController
class RAGController {
    private final ChatClient chatModel;
    private final VectorStore vectorStore;

    RAGController(ChatClient chatModel, VectorStore vectorStore) {
        this.chatModel = chatModel;
        this.vectorStore = vectorStore;
    }

    @GetMapping("/ai/rag/people")
    Person chatWithRag(@RequestParam String name) {
        // 步骤1：使用自然语言查询 VectorStore，查找相似文档
        List<Document> similarDocuments = vectorStore.similaritySearch(
                SearchRequest.builder()
                    .query(name)
                    .topK(2)
                    .build());
        
        // 步骤2：将相似文档内容拼接为上下文信息
        String information = similarDocuments.stream()
                .map(Document::getText)
                .collect(Collectors.joining(System.lineSeparator()));

        // 步骤3：构建 SystemPromptTemplate，指示模型使用提供的上下文信息回答问题
        var systemPromptTemplate = new SystemPromptTemplate("""
              You are a helpful assistant.
              
              Use the following information to answer the question:
              {information}
              """);
        var systemMessage = systemPromptTemplate.createMessage(
                Map.of("information", information));

        // 步骤4：使用 BeanOutputConverter 将响应解析为结构化对象
        var outputParser = new BeanOutputConverter<>(Person.class);

        // 步骤5：构建用户消息，要求模型介绍指定人物
        PromptTemplate userMessagePromptTemplate = new PromptTemplate("""
        Tell me about {name} as if current date is {current_date}.

        {format}
        """);
        Map<String,Object> model = Map.of("name", name,
                "current_date", LocalDate.now(),
                "format", outputParser.getFormat());
        var userMessage = new UserMessage(userMessagePromptTemplate.create(model).getContents());

        // 步骤6：发送 Prompt 并获取响应
        var prompt = new Prompt(List.of(systemMessage, userMessage));
        String response = chatModel.prompt(prompt).call().content();

        // 步骤7：将响应转换为结构化对象并返回
        return outputParser.convert(response);
    }
}

// 定义响应结构
record Person(String name,
              String dateOfBirth,
              int experienceInYears,
              List<String> books) {
}
```

完整的代码可参考 [RAGController.java](file:///home/wandl/workspaces/workspace-partme-ai/spring-ai-examples/spring-ai-ollama-rag-cassandra/src/main/java/com/github/teachingai/ollama/controller/RAGController.java)。

## 最佳实践

### 1. 向量维度匹配

Cassandra 向量存储需要配置正确的向量维度。应确保嵌入模型生成的向量维度与配置一致：

```properties
spring.ai.vectorstore.cassandra.vector-dimensionality=768
```

不同的嵌入模型有不同的维度：
- `mistral`: 768 维
- `nomic-embed-text`: 768 维
- `mxbai-embed-large`: 1024 维

### 2. 批量操作优化

对于大量文档的添加，建议使用批量操作：

```java
public void addDocumentsBatch(List<Document> documents) {
    // 分批处理，每批 100 条
    List<List<Document>> batches = ListUtils.partition(documents, 100);
    for (List<Document> batch : batches) {
        vectorStore.add(batch);
    }
}
```

### 3. 索引优化

Cassandra 向量搜索性能依赖于索引配置。根据数据量调整搜索参数：

```java
SearchRequest.builder()
    .query(query)
    .topK(10)  // 根据准确率需求调整
    .similarityThreshold(0.7)  // 设置相似度阈值
    .build();
```

## 故障排除

### 常见问题

1. **无法连接 Cassandra**
   - 检查 Cassandra 服务是否启动
   - 确认端口 9042 可访问
   - 验证认证配置（如有）

2. **嵌入失败**
   - 确认 Ollama 已拉取 `application.properties` 中配置的嵌入模型
   - 检查 Ollama 服务是否正常运行
   - 验证模型名称是否正确

3. **向量维度不匹配**
   - 检查嵌入模型输出的向量维度
   - 确保 Cassandra 配置中的维度与模型一致

4. **查询性能问题**
   - 考虑增加 Cassandra 节点以提高吞吐量
   - 优化向量索引配置
   - 调整 topK 参数减少返回结果数量
