# Spring AI 入门实践：Spring AI 与 GemFire 向量数据库（Vector Databases）

> 基于 Spring AI 框架实现与 GemFire 向量数据库的集成，结合分布式内存数据平台和近似最近邻搜索能力，提供文档向量存储、相似度搜索、低延迟访问等功能。

## 二、GemFire 向量数据库简介

### 2.1 GemFire 介绍

Apache Geode（现称为 VMware Tanzu GemFire）是一个高性能的分布式内存数据管理平台，专为大规模应用提供低延迟的数据访问。GemFire 支持将向量数据存储在分布式缓存中，并提供近似最近邻（ANN）搜索功能。

### 2.2 核心特性

| 特性 | 说明 |
|------|------|
| **分布式缓存** | 分布式内存数据管理 |
| **低延迟** | 毫秒级响应时间 |
| **ANN 搜索** | 近似最近邻搜索 |
| **水平扩展** | 支持集群水平扩展 |
| **事务支持** | ACID 事务支持 |
| **高可用** | 自动故障转移 |

### 2.3 应用场景

- 金融交易系统
- 实时分析
- 游戏排行榜
- 缓存加速

## 完整示例

### 向量存储基本操作

以下示例展示了使用 VectorStore 进行文档存储和相似度搜索：

```java
import org.springframework.ai.document.Document;
import org.springframework.ai.vectorstore.SearchRequest;
import org.springframework.ai.vectorstore.VectorStore;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

import java.util.List;
import java.util.Map;

@Service
public class VectorStoreService {

    @Autowired
    private VectorStore vectorStore;

    public void addDocuments() {
        List<Document> documents = List.of(
            new Document("GemFire 是一个分布式内存数据平台", Map.of("category", "database")),
            new Document("Spring AI 提供了 AI 能力集成", Map.of("category", "ai")),
            new Document("向量搜索支持近似最近邻搜索", Map.of("category", "vector")),
            new Document("RAG 技术结合了检索和生成", Map.of("category", "ai"))
        );
        
        vectorStore.add(documents);
        System.out.println("文档添加完成");
    }

    public List<Document> searchDocuments(String query) {
        return vectorStore.similaritySearch(
            SearchRequest.builder()
                .query(query)
                .topK(3)
                .build()
        );
    }
}
```

### 创建 GemFire 区域

需要配置 GemFire 区域来存储向量数据：

```java
@Bean
public Region<String, Document> vectorRegion(GemFireCache gemFireCache) {
    return gemFireCache.<String, Document>createRegionFactory(RegionShortcut.PARTITION)
        .setKeyConstraint(String.class)
        .setValueConstraint(Document.class)
        .create("vector_store");
}
```

### RAG 实现示例

以下是一个完整的 RAG 实现示例：

```java
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.chat.messages.UserMessage;
import org.springframework.ai.chat.prompt.Prompt;
import org.springframework.ai.chat.prompt.PromptTemplate;
import org.springframework.ai.chat.prompt.SystemPromptTemplate;
import org.springframework.ai.document.Document;
import org.springframework.ai.vectorstore.SearchRequest;
import org.springframework.ai.vectorstore.VectorStore;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;

import java.util.List;
import java.util.Map;
import java.util.stream.Collectors;

@RestController
class RAGController {
    private final ChatClient chatModel;
    private final VectorStore vectorStore;

    RAGController(ChatClient chatModel, VectorStore vectorStore) {
        this.chatModel = chatModel;
        this.vectorStore = vectorStore;
    }

    @GetMapping("/ai/rag/search")
    String searchWithRag(@RequestParam String query) {
        // 步骤1：使用自然语言查询 VectorStore
        List<Document> similarDocuments = vectorStore.similaritySearch(
                SearchRequest.builder()
                    .query(query)
                    .topK(3)
                    .build());
        
        // 步骤2：将相似文档内容拼接为上下文信息
        String context = similarDocuments.stream()
                .map(Document::getText)
                .collect(Collectors.joining("\n\n"));

        // 步骤3：构建 System Prompt
        var systemPromptTemplate = new SystemPromptTemplate("""
              你是一个有帮助的助手。
              请根据以下信息回答用户的问题：
              {context}
              """);
        var systemMessage = systemPromptTemplate.createMessage(
                Map.of("context", context));

        // 步骤4：构建用户消息
        var userMessage = new UserMessage(query);

        // 步骤5：发送 Prompt 并获取响应
        var prompt = new Prompt(List.of(systemMessage, userMessage));
        return chatModel.prompt(prompt).call().content();
    }
}
```

## 最佳实践

### 1. 集群配置优化

GemFire 的性能依赖于集群配置：

```properties
spring.geode.locators=host1[10334],host2[10334]
spring.geode.pool.locators=host1[10334],host2[10334]
spring.geode.cache.server.bind-address=0.0.0.0
spring.geode.cache.server.port=40404
```

### 2. 区域配置优化

配置高可用性和数据分区：

```java
@Bean
public RegionFactory<String, Document> regionFactory(GemFireCache gemFireCache) {
    return gemFireCache.<String, Document>createRegionFactory(RegionShortcut.PARTITION_REDUNDANT);
}
```

### 3. 批量操作优化

对于大量文档的添加，建议使用批量操作：

```java
public void addDocumentsBatch(List<Document> documents) {
    // 分批处理，每批 100 条
    List<List<Document>> batches = ListUtils.partition(documents, 100);
    for (List<Document> batch : batches) {
        vectorStore.add(batch);
    }
}
```

## 故障排除

### 常见问题

1. **无法连接 GemFire**
   - 检查 GemFire 服务是否启动
   - 确认_locator 端口 10334 可访问
   - 验证集群配置正确

2. **区域创建失败**
   - 检查 GemFire 集群状态
   - 验证区域配置正确
   - 检查磁盘存储配置

3. **嵌入失败**
   - 确认 Ollama 已拉取配置的嵌入模型
   - 检查 Ollama 服务是否正常运行
   - 验证模型名称是否正确

4. **搜索性能问题**
   - 调整 GemFire 缓存配置
   - 优化数据分区策略
   - 考虑使用内存优化配置

## 六、Java 客户端示例

### 6.1 REST 客户端

```java
import org.springframework.web.client.RestTemplate;
import org.springframework.http.HttpEntity;
import org.springframework.http.HttpHeaders;
import org.springframework.http.MediaType;
import java.util.List;
import java.util.Map;

public class GemFireClient {

    private static final String BASE_URL = "http://localhost:8080/api/gemfire";
    private final RestTemplate restTemplate = new RestTemplate();

    public void addDocuments(List<Map<String, Object>> docs) {
        HttpHeaders headers = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_JSON);
        HttpEntity<List<Map<String, Object>>> request = new HttpEntity<>(docs, headers);
        restTemplate.postForObject(BASE_URL + "/documents", request, String.class);
    }

    public List<Map<String, Object>> search(String query, int topK) {
        String url = BASE_URL + "/search?query={query}&topK={topK}";
        return restTemplate.getForObject(url, List.class, query, topK);
    }
}
```
